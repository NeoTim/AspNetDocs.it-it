### <a name="url-patterns"></a><span data-ttu-id="99ada-131">Modelli di URL</span><span class="sxs-lookup"><span data-stu-id="99ada-131">URL Patterns</span></span>

<span data-ttu-id="99ada-132">Un modello di URL può contenere valori letterali e segnaposto di variabile (detti parametri URL).</span><span class="sxs-lookup"><span data-stu-id="99ada-132">A URL pattern can contain literal values and variable placeholders (referred to as URL parameters).</span></span> <span data-ttu-id="99ada-133">I valori letterali e i segnaposto si trovano in segmenti dell'URL delimitati dal carattere barra (`/`).</span><span class="sxs-lookup"><span data-stu-id="99ada-133">The literals and placeholders are located in segments of the URL which are delimited by the slash (`/`) character.</span></span>

<span data-ttu-id="99ada-134">Quando viene effettuata una richiesta all'applicazione Web, l'URL viene analizzato in segmenti e segnaposto e i valori delle variabili vengono forniti al gestore della richiesta.</span><span class="sxs-lookup"><span data-stu-id="99ada-134">When a request to your web application is made, the URL is parsed into segments and placeholders, and the variable values are provided to the request handler.</span></span> <span data-ttu-id="99ada-135">Questo processo è simile al modo in cui i dati di una stringa di query vengono analizzati e passati al gestore della richiesta.</span><span class="sxs-lookup"><span data-stu-id="99ada-135">This process is similar to the way the data in a query string is parsed and passed to the request handler.</span></span> <span data-ttu-id="99ada-136">In entrambi i casi, le informazioni sulle variabili sono incluse nell'URL e passate al gestore sotto forma di coppie chiave-valore.</span><span class="sxs-lookup"><span data-stu-id="99ada-136">In both cases, variable information is included in the URL and passed to the handler in the form of key-value pairs.</span></span> <span data-ttu-id="99ada-137">Per le stringhe di query, le chiavi e i valori sono inclusi nell'URL.</span><span class="sxs-lookup"><span data-stu-id="99ada-137">For query strings, both the keys and the values are in the URL.</span></span> <span data-ttu-id="99ada-138">Per le route, le chiavi sono i nomi di segnaposto definiti nel modello di URL e solo i valori sono presenti nell'URL.</span><span class="sxs-lookup"><span data-stu-id="99ada-138">For routes, the keys are the placeholder names defined in the URL pattern, and only the values are in the URL.</span></span>
