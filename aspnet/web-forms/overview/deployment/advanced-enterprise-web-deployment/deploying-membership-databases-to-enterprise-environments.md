## <a name="what-are-the-issues-when-you-deploy-a-membership-database"></a><span data-ttu-id="860df-111">Quali sono i problemi quando si distribuisce un database di appartenenza?</span><span class="sxs-lookup"><span data-stu-id="860df-111">What Are the Issues When You Deploy a Membership Database?</span></span>

<span data-ttu-id="860df-112">Nella maggior parte dei casi, quando si progetta una strategia di distribuzione per un database, è prima di tutto necessario considerare i dati che si desidera distribuire.</span><span class="sxs-lookup"><span data-stu-id="860df-112">In most cases, when you devise a deployment strategy for a database, the first thing you need to consider is what data you want to deploy.</span></span> <span data-ttu-id="860df-113">In un ambiente di sviluppo o di test, è possibile distribuire i dati dell'account utente per facilitare i test rapidi e semplici.</span><span class="sxs-lookup"><span data-stu-id="860df-113">In a development or test environment, you might want to deploy user account data to facilitate quick and easy testing.</span></span> <span data-ttu-id="860df-114">In un ambiente di gestione temporanea o di produzione, è molto improbabile che si desideri distribuire i dati dell'account utente.</span><span class="sxs-lookup"><span data-stu-id="860df-114">In a staging or production environment, it's very unlikely that you'd want to deploy user account data.</span></span>

<span data-ttu-id="860df-115">Sfortunatamente, i database di appartenenza a ASP.NET introducono alcune difficoltà specifiche che rendono questa decisione molto più complessa:</span><span class="sxs-lookup"><span data-stu-id="860df-115">Unfortunately, ASP.NET membership databases introduce some specific challenges that make this decision a lot more complex:</span></span>

- <span data-ttu-id="860df-116">Una distribuzione di solo schema lascerà il database di appartenenza in uno stato non operativo.</span><span class="sxs-lookup"><span data-stu-id="860df-116">A schema-only deployment will leave the membership database in a non-operational state.</span></span> <span data-ttu-id="860df-117">Questo è dovuto al fatto che il database di appartenenza include alcuni dati di configurazione (nella tabella **aspnet\_SchemaVersions** ) necessari per il funzionamento del database.</span><span class="sxs-lookup"><span data-stu-id="860df-117">This is because the membership database includes some configuration data (in the **aspnet\_SchemaVersions** table) that the database requires in order to function.</span></span> <span data-ttu-id="860df-118">Di conseguenza, se si esegue una distribuzione solo schema del database di appartenenza per escludere i dati dell'account utente, è necessario eseguire uno script di post-distribuzione per aggiungere i dati di configurazione essenziali.</span><span class="sxs-lookup"><span data-stu-id="860df-118">As such, if you perform a schema-only deployment of your membership database in order to exclude user account data, you'll need to run a post-deployment script to add the essential configuration data.</span></span>
- <span data-ttu-id="860df-119">A seconda di come viene configurato il database di appartenenza, il provider di appartenenze può utilizzare la chiave del computer per crittografare le password e archiviarle nel database.</span><span class="sxs-lookup"><span data-stu-id="860df-119">Depending on how your membership database is configured, the membership provider may use the machine key to encrypt passwords and store them in the database.</span></span> <span data-ttu-id="860df-120">In questo caso, tutti i dati dell'account utente distribuiti con il database diventeranno inutilizzabili nel server di destinazione.</span><span class="sxs-lookup"><span data-stu-id="860df-120">In this case, any user account data you deploy with the database will become unusable on the destination server.</span></span> <span data-ttu-id="860df-121">Per questo motivo, la distribuzione dei dati dell'account utente non è uno scenario supportato.</span><span class="sxs-lookup"><span data-stu-id="860df-121">For this reason, deploying user account data is not a supported scenario.</span></span>

## <a name="choosing-a-membership-database-strategy"></a><span data-ttu-id="860df-122">Scelta di una strategia di database di appartenenza</span><span class="sxs-lookup"><span data-stu-id="860df-122">Choosing a Membership Database Strategy</span></span>

<span data-ttu-id="860df-123">Utilizzare queste linee guida quando si sceglie come effettuare il provisioning di un database di appartenenza in un ambiente Enterprise Server:</span><span class="sxs-lookup"><span data-stu-id="860df-123">Use these guidelines when you choose how to provision a membership database in an enterprise server environment:</span></span>

- <span data-ttu-id="860df-124">Laddove possibile, non distribuire i database di appartenenza.</span><span class="sxs-lookup"><span data-stu-id="860df-124">Wherever possible, do not deploy membership databases.</span></span> <span data-ttu-id="860df-125">Al contrario, creare manualmente il database delle appartenenze nel server di database di destinazione.</span><span class="sxs-lookup"><span data-stu-id="860df-125">Instead, create the membership database manually on the target database server.</span></span> <span data-ttu-id="860df-126">Se lo schema del database di appartenenza non è stato personalizzato, è sufficiente crearne uno nuovo in situ nella destinazione usando lo [strumento di registrazione ASP.NET SQL Server (aspnet\_RegSql. exe)](https://msdn.microsoft.com/library/ms229862(v=vs.100).aspx).</span><span class="sxs-lookup"><span data-stu-id="860df-126">If you haven't customized your membership database schema, you can simply create a new one in situ at the destination using the [ASP.NET SQL Server Registration Tool (aspnet\_regsql.exe)](https://msdn.microsoft.com/library/ms229862(v=vs.100).aspx).</span></span>
- <span data-ttu-id="860df-127">Se non si dispone di un'opzione ma si distribuisce&#x2014;un database di appartenenza, ad esempio se sono state apportate&#x2014;modifiche estese allo schema del database, è consigliabile eseguire una distribuzione solo schema del database di appartenenza, per escludere i dati dell'account utente, quindi eseguire uno script di post-distribuzione per aggiungere i dati di configurazione necessari.</span><span class="sxs-lookup"><span data-stu-id="860df-127">If you have no option but to deploy a membership database&#x2014;for example, if you've made extensive modifications to the database schema&#x2014;you should perform a schema-only deployment of the membership database, to exclude user account data, and then run a post-deployment script to add any required configuration data.</span></span> <span data-ttu-id="860df-128">Per informazioni generali su questi approcci [, vedere Procedura: distribuire il database delle appartenenze di ASP.NET senza includere gli account utente](https://msdn.microsoft.com/library/ff361972(v=vs.100).aspx).</span><span class="sxs-lookup"><span data-stu-id="860df-128">You can find broad guidance on these approaches in [How to: Deploy the ASP.NET Membership Database Without Including User Accounts](https://msdn.microsoft.com/library/ff361972(v=vs.100).aspx).</span></span>

<span data-ttu-id="860df-129">È importante ricordare che *lo schema del database di appartenenza è probabilmente piuttosto statico*.</span><span class="sxs-lookup"><span data-stu-id="860df-129">It's important to remember that *the schema of your membership database is likely to be fairly static*.</span></span> <span data-ttu-id="860df-130">Anche se il database di appartenenza è stato personalizzato, è improbabile che sia necessario aggiornare lo schema a intervalli regolari,&#x2014;ma non cambierà con la stessa frequenza del codice in un'applicazione Web o in un progetto di database.</span><span class="sxs-lookup"><span data-stu-id="860df-130">Even if you've customized the membership database, it's unlikely that you'll need to update the schema on a regular basis&#x2014;it's not going to change with the same frequency as the code in a web application or a database project.</span></span> <span data-ttu-id="860df-131">Di conseguenza, non è necessario includere il database di appartenenza in tutti i processi di distribuzione automatizzati o in un singolo passaggio.</span><span class="sxs-lookup"><span data-stu-id="860df-131">As such, you shouldn't need to include the membership database in any automated or single-step deployment processes.</span></span>

<span data-ttu-id="860df-136">In alternativa, è possibile utilizzare l'utilità VSDBCMD per aggiornare lo schema del database di destinazione.</span><span class="sxs-lookup"><span data-stu-id="860df-136">The alternative is to use the VSDBCMD utility to update the schema of your destination database.</span></span> <span data-ttu-id="860df-137">VSDBCMD include due funzionalità importanti.</span><span class="sxs-lookup"><span data-stu-id="860df-137">VSDBCMD includes two important capabilities.</span></span> <span data-ttu-id="860df-138">In primo luogo, è possibile importare lo schema di un database esistente in un file con estensione dbschema.</span><span class="sxs-lookup"><span data-stu-id="860df-138">First, it can import the schema of an existing database into a .dbschema file.</span></span> <span data-ttu-id="860df-139">In secondo luogo, è in grado di distribuire un file con estensione dbschema a un database esistente come aggiornamento differenziale, il che significa che apporta solo le modifiche necessarie per portare aggiornato il database di destinazione e non si perdono i dati.</span><span class="sxs-lookup"><span data-stu-id="860df-139">Second, it can deploy a .dbschema file to an existing database as a differential update, which means that it only makes the changes required to bring the target database up to date and you don't lose any data.</span></span>

2. <span data-ttu-id="860df-143">Usare l'azione di **distribuzione** VSDBCMD per distribuire il file con estensione dbschema nel database delle appartenenze di destinazione.</span><span class="sxs-lookup"><span data-stu-id="860df-143">Use the VSDBCMD **Deploy** action to deploy the .dbschema file to your destination membership database.</span></span> <span data-ttu-id="860df-144">Questa procedura è descritta in [riferimento alla riga di comando per VSDBCMD.
