## <a name="what-are-the-issues-when-you-deploy-a-membership-database"></a><span data-ttu-id="4ed23-111">Quali sono i problemi quando si distribuisce un Database di appartenenza?</span><span class="sxs-lookup"><span data-stu-id="4ed23-111">What Are the Issues When You Deploy a Membership Database?</span></span>

<span data-ttu-id="4ed23-112">Nella maggior parte dei casi, quando si escogitare una strategia di distribuzione per un database, la prima cosa da considerare è quali dati si desidera distribuire.</span><span class="sxs-lookup"><span data-stu-id="4ed23-112">In most cases, when you devise a deployment strategy for a database, the first thing you need to consider is what data you want to deploy.</span></span> <span data-ttu-id="4ed23-113">In un ambiente di sviluppo o test, si potrebbe essere necessario distribuire i dati dell'account utente per facilitare il test rapido e immediato.</span><span class="sxs-lookup"><span data-stu-id="4ed23-113">In a development or test environment, you might want to deploy user account data to facilitate quick and easy testing.</span></span> <span data-ttu-id="4ed23-114">In un ambiente di gestione temporanea o produzione, è molto improbabile che si desidera distribuire i dati dell'account utente.</span><span class="sxs-lookup"><span data-stu-id="4ed23-114">In a staging or production environment, it's very unlikely that you'd want to deploy user account data.</span></span>

<span data-ttu-id="4ed23-115">Sfortunatamente, i database di appartenenza ASP.NET introducono alcuni problemi specifici che prendere questa decisione molto più complessa:</span><span class="sxs-lookup"><span data-stu-id="4ed23-115">Unfortunately, ASP.NET membership databases introduce some specific challenges that make this decision a lot more complex:</span></span>

- <span data-ttu-id="4ed23-116">Una distribuzione con solo schema lascia il database delle appartenenze in uno stato non operativo.</span><span class="sxs-lookup"><span data-stu-id="4ed23-116">A schema-only deployment will leave the membership database in a non-operational state.</span></span> <span data-ttu-id="4ed23-117">Infatti, il database di appartenenza comprende alcuni dati di configurazione (nelle **aspnet\_SchemaVersions** tabella) che il database necessita per funzionare.</span><span class="sxs-lookup"><span data-stu-id="4ed23-117">This is because the membership database includes some configuration data (in the **aspnet\_SchemaVersions** table) that the database requires in order to function.</span></span> <span data-ttu-id="4ed23-118">Di conseguenza, se si esegue una distribuzione solo allo schema del database di appartenenza per escludere i dati dell'account utente, è necessario eseguire uno script di post-distribuzione e aggiungere i dati di configurazione essenziali.</span><span class="sxs-lookup"><span data-stu-id="4ed23-118">As such, if you perform a schema-only deployment of your membership database in order to exclude user account data, you'll need to run a post-deployment script to add the essential configuration data.</span></span>
- <span data-ttu-id="4ed23-119">A seconda del modo in cui il database di appartenenze è configurato, il provider di appartenenze può usare la chiave del computer per crittografare le password e archiviarli nel database.</span><span class="sxs-lookup"><span data-stu-id="4ed23-119">Depending on how your membership database is configured, the membership provider may use the machine key to encrypt passwords and store them in the database.</span></span> <span data-ttu-id="4ed23-120">In questo caso, eventuali dati dell'account utente che si distribuisce con il database diventerà inutilizzabili nel server di destinazione.</span><span class="sxs-lookup"><span data-stu-id="4ed23-120">In this case, any user account data you deploy with the database will become unusable on the destination server.</span></span> <span data-ttu-id="4ed23-121">Per questo motivo, la distribuzione di dati dell'account utente non è uno scenario supportato.</span><span class="sxs-lookup"><span data-stu-id="4ed23-121">For this reason, deploying user account data is not a supported scenario.</span></span>

## <a name="choosing-a-membership-database-strategy"></a><span data-ttu-id="4ed23-122">Scelta di una strategia di Database di appartenenza</span><span class="sxs-lookup"><span data-stu-id="4ed23-122">Choosing a Membership Database Strategy</span></span>

<span data-ttu-id="4ed23-123">Usare queste linee guida quando si sceglie come effettuare il provisioning di un database di appartenenza in un ambiente server aziendale:</span><span class="sxs-lookup"><span data-stu-id="4ed23-123">Use these guidelines when you choose how to provision a membership database in an enterprise server environment:</span></span>

- <span data-ttu-id="4ed23-124">Laddove possibile, non vengono distribuiti i database di appartenenza.</span><span class="sxs-lookup"><span data-stu-id="4ed23-124">Wherever possible, do not deploy membership databases.</span></span> <span data-ttu-id="4ed23-125">In alternativa, creare manualmente i database delle appartenenze nel server di database di destinazione.</span><span class="sxs-lookup"><span data-stu-id="4ed23-125">Instead, create the membership database manually on the target database server.</span></span> <span data-ttu-id="4ed23-126">Se non è stato personalizzato dello schema del database di appartenenza, è possibile semplicemente creare uno nuovo in situ, di destinazione utilizzando il [strumento di registrazione di SQL Server ASP.NET (aspnet\_regsql.exe)](https://msdn.microsoft.com/library/ms229862(v=vs.100).aspx).</span><span class="sxs-lookup"><span data-stu-id="4ed23-126">If you haven't customized your membership database schema, you can simply create a new one in situ at the destination using the [ASP.NET SQL Server Registration Tool (aspnet\_regsql.exe)](https://msdn.microsoft.com/library/ms229862(v=vs.100).aspx).</span></span>
- <span data-ttu-id="4ed23-127">Se non esiste nessuna opzione ma alla distribuzione di un database di appartenenza&#x2014;, ad esempio, se sono state apportate modifiche estese allo schema del database&#x2014;è consigliabile eseguire una distribuzione di solo schema di database di appartenenza, per escludere i dati dell'account utente e quindi eseguire uno script di post-distribuzione per aggiungere tutti i dati di configurazione necessarie.</span><span class="sxs-lookup"><span data-stu-id="4ed23-127">If you have no option but to deploy a membership database&#x2014;for example, if you've made extensive modifications to the database schema&#x2014;you should perform a schema-only deployment of the membership database, to exclude user account data, and then run a post-deployment script to add any required configuration data.</span></span> <span data-ttu-id="4ed23-128">È possibile trovare indicazioni generali su questi approcci in [come: Distribuire il Database di appartenenza ASP.NET senza includere gli account utente](https://msdn.microsoft.com/library/ff361972(v=vs.100).aspx).</span><span class="sxs-lookup"><span data-stu-id="4ed23-128">You can find broad guidance on these approaches in [How to: Deploy the ASP.NET Membership Database Without Including User Accounts](https://msdn.microsoft.com/library/ff361972(v=vs.100).aspx).</span></span>

<span data-ttu-id="4ed23-129">È importante ricordare che *lo schema del database di appartenenza è probabile che sia abbastanza statici*.</span><span class="sxs-lookup"><span data-stu-id="4ed23-129">It's important to remember that *the schema of your membership database is likely to be fairly static*.</span></span> <span data-ttu-id="4ed23-130">Anche se è stato personalizzato il database di appartenenza, è improbabile che è necessario aggiornare lo schema a intervalli regolari&#x2014;non verrà modificata con la stessa frequenza con cui il codice in un'applicazione web o un progetto di database.</span><span class="sxs-lookup"><span data-stu-id="4ed23-130">Even if you've customized the membership database, it's unlikely that you'll need to update the schema on a regular basis&#x2014;it's not going to change with the same frequency as the code in a web application or a database project.</span></span> <span data-ttu-id="4ed23-131">Di conseguenza, non è necessario includere il database delle appartenenze in tutti i processi automatizzati o passo a passo la distribuzione.</span><span class="sxs-lookup"><span data-stu-id="4ed23-131">As such, you shouldn't need to include the membership database in any automated or single-step deployment processes.</span></span>

<span data-ttu-id="4ed23-136">L'alternativa consiste nell'usare l'utilità VSDBCMD per aggiornare lo schema del database di destinazione.</span><span class="sxs-lookup"><span data-stu-id="4ed23-136">The alternative is to use the VSDBCMD utility to update the schema of your destination database.</span></span> <span data-ttu-id="4ed23-137">VSDBCMD include due funzionalità importanti.</span><span class="sxs-lookup"><span data-stu-id="4ed23-137">VSDBCMD includes two important capabilities.</span></span> <span data-ttu-id="4ed23-138">In primo luogo, è possibile importare lo schema di un database esistente in un file. dbschema.</span><span class="sxs-lookup"><span data-stu-id="4ed23-138">First, it can import the schema of an existing database into a .dbschema file.</span></span> <span data-ttu-id="4ed23-139">In secondo luogo, è possibile distribuire un file di dbschema a un database esistente come un aggiornamento differenziale, il che significa che esegue solo le modifiche necessarie per portare il database di destinazione aggiornato e non perdere i dati.</span><span class="sxs-lookup"><span data-stu-id="4ed23-139">Second, it can deploy a .dbschema file to an existing database as a differential update, which means that it only makes the changes required to bring the target database up to date and you don't lose any data.</span></span>
